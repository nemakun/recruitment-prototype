# 採用管理WEBアプリ 基本設計書（初版）

## 1. 文書情報
- 文書名: 採用管理WEBアプリ 基本設計書
- 版数: v0.1
- 作成日: 2026-02-09
- 対象システム: 採用管理WEBアプリ（プロトタイプ）

## 2. 目的
- 採用業務（応募受付、面接進行、評価、意思決定、分析）を一元管理する。
- 採用担当者・面接官・部門責任者・技術部門担当者が、役割に応じて同一基盤上で運用できる状態にする。
- ダッシュボードにより、応募数・通過率・所要日数を可視化し、前年比・前年同期比・推移分析を可能にする。

## 3. 適用範囲（スコープ）
### 3.1 対象
- WEBアプリケーション（フロントエンド + バックエンドAPI）
- ログイン認証（メール/パスワード）
- 候補者管理、選考管理、管理者機能、ダッシュボード分析

### 3.2 対象外（現時点）
- SSO認証
- 外部採用媒体連携（リクナビ等）
- CSV一括取込
- 本番DB永続化、厳格監査基盤

## 4. 利用者・ロール
- 採用担当（recruiter）: 3名想定
- 面接官（interviewer）: 16名想定（グループリーダー/課長）
- 部門責任者（dept_manager）: 2名想定（各部部長）
- 技術部門担当者（tech_admin）: 2名想定

## 5. 組織モデル
- 2部（第一開発部、第二開発部）
- 各部2課（第一課、第二課）
- 各課3グループ（A/B/C）
- 採用単位: グループ

## 6. システム構成
- フロントエンド: React + Vite
- バックエンド: Express + TypeScript
- 配備構成:
  - Vercel（フロント）
  - Render（バックエンド）

## 7. 機能一覧
### 7.1 認証
- メール/パスワードログイン
- 初期パスワード運用（`rec12345`）
- 退職者はアカウント無効化

### 7.2 ダッシュボード
- KPI: 応募数、通過率、所要日数（中央値）
- 集計期間: 月次 / 四半期 / 半期 / 年次
- 絞り込み条件: 対象月、対象四半期、対象半期、対象年度（過去3年）
- 年度設定連動: 4月始まり / 1月始まり / 9月始まり
- 分析:
  - 前期比、前年同期比（全体）
  - 3年推移（全体）
  - 前年同期比較テーブル（部/課/グループ）
- 分布:
  - 応募数分布（部/課/グループ）
  - 通過率分布（部/課/グループ）

### 7.3 候補者管理
- 候補者一覧（表形式）
- 新規登録/編集
- 検索（氏名、メール、部門、ステータス）
- ソート（応募日ほか）
- ページング（先頭/前へ/次へ/末尾）

### 7.4 選考管理
- 応募情報（応募日、応募先部課グループ）
- 一次面接（実施日、担当者最大2名、合否、コメント、回答日）
- 最終面接（実施日、担当者最大2名、合否、コメント、回答日）
- 面接官コメントは必須、採用担当コメントは任意（運用要件）

### 7.5 管理者機能
- アカウント追加/編集
- 年度設定変更
- 管理者画面アクセス制御（recruiter/tech_adminのみ）

## 8. 権限設計（概要）
- recruiter:
  - 管理者画面アクセス可
  - ステータス変更可
- tech_admin:
  - 管理者画面アクセス可
  - ステータス変更不可
- dept_manager:
  - 管理者画面アクセス不可
  - ステータス変更可
- interviewer:
  - 管理者画面アクセス不可
  - ステータス変更不可

## 9. データ保持・監査（方針）
- 候補者データ保持: 10年
- 監査ログ保持: 15年
- 個人情報ポリシー（仮）: 論理削除後、10年経過で匿名化

## 10. 非機能要件（初期）
- 可用性: 無償クラウド範囲での提供（Render無料枠スリープあり）
- 性能: 2,000件規模の一覧操作に耐える応答
- セキュリティ: プロトタイプ段階（本番前に強化必須）
- 拡張性: 集計項目・分析軸を追加しやすい設計を維持

## 11. 画面一覧
- ログイン画面
- ダッシュボード画面
- 候補者一覧画面
- 候補者新規登録画面
- 候補者編集画面
- 管理者画面

## 12. 画面遷移（概要）
- ログイン成功 → ダッシュボード
- ハンバーガーメニューで各画面へ遷移
- 新規登録/編集完了 → 候補者一覧へ戻る
- 管理者権限なしで管理者画面遷移時:
  - エラー表示
  - ダッシュボードへリダイレクト
